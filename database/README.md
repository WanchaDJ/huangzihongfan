# 黄子弘凡个人网站数据库设计

## 项目概述
这是黄子弘凡个人官方网站的数据库设计，包含用户管理、会员系统、音乐专辑、照片画廊、新闻资讯、商品销售、订单管理等核心功能模块。

## 数据库结构

### 核心表结构

#### 1. 用户表 (users)
- 存储网站用户的基本信息
- 包含用户名、邮箱、手机号、密码等字段
- 支持用户登录、注册、个人信息管理等功能

#### 2. 会员等级表 (member_levels)
- 定义会员等级体系
- 包含等级名称、积分要求、等级特权等字段
- 支持会员等级晋升、特权管理等功能

#### 3. 会员表 (members)
- 存储用户会员信息
- 包含用户ID、当前等级、积分、会员有效期等字段
- 支持会员资格管理、积分累积等功能

#### 4. 专辑表 (albums)
- 存储音乐专辑信息
- 包含专辑标题、封面、发行日期、音乐类型等字段
- 支持专辑展示、浏览等功能

#### 5. 歌曲表 (songs)
- 存储专辑中的歌曲信息
- 包含歌曲标题、音频URL、歌词、时长等字段
- 支持歌曲播放、下载等功能

#### 6. 照片画廊分类表 (gallery_categories)
- 定义照片画廊的分类体系
- 包含分类名称、描述、排序等字段
- 支持照片分类管理、浏览等功能

#### 7. 照片表 (photos)
- 存储照片信息
- 包含照片标题、图片URL、缩略图、拍摄日期等字段
- 支持照片展示、浏览、下载等功能

#### 8. 新闻分类表 (news_categories)
- 定义新闻资讯的分类体系
- 包含分类名称、描述、排序等字段
- 支持新闻分类管理、浏览等功能

#### 9. 新闻表 (news)
- 存储新闻资讯信息
- 包含新闻标题、内容、封面、作者、来源等字段
- 支持新闻展示、浏览、评论等功能

#### 10. 商品分类表 (product_categories)
- 定义商品的分类体系
- 包含分类名称、父分类ID、描述、排序等字段
- 支持商品分类管理、浏览等功能

#### 11. 商品表 (products)
- 存储商品信息
- 包含商品标题、描述、图片、价格、库存等字段
- 支持商品展示、浏览、购买等功能

#### 12. 订单表 (orders)
- 存储用户订单信息
- 包含订单编号、用户ID、订单金额、支付方式、订单状态等字段
- 支持订单管理、支付处理、发货管理等功能

#### 13. 订单商品表 (order_items)
- 存储订单中的商品信息
- 包含订单ID、商品ID、商品标题、价格、数量等字段
- 支持订单商品明细管理、统计等功能

#### 14. 购物车表 (cart_items)
- 存储用户购物车信息
- 包含用户ID、商品ID、商品标题、价格、数量等字段
- 支持购物车管理、商品添加、删除等功能

#### 15. 积分记录表 (points_logs)
- 存储用户积分变化记录
- 包含用户ID、积分数量、变化类型、相关ID等字段
- 支持积分管理、积分变更记录查询等功能

#### 16. 评论表 (comments)
- 存储用户评论信息
- 包含用户ID、评论目标类型、目标ID、评论内容、父评论ID等字段
- 支持评论管理、评论回复、点赞等功能

#### 17. 系统配置表 (system_configs)
- 存储系统配置信息
- 包含配置键、配置值、配置类型、描述等字段
- 支持系统配置管理、动态配置更新等功能

## 数据库关系图

### 表关系说明
1. **用户与会员**：一个用户可以有一个会员信息（一对一关系）
2. **会员与会员等级**：一个会员属于一个会员等级，一个会员等级可以包含多个会员（一对多关系）
3. **专辑与歌曲**：一个专辑包含多个歌曲，一个歌曲属于一个专辑（一对多关系）
4. **照片画廊分类与照片**：一个分类包含多个照片，一个照片属于一个分类（一对多关系）
5. **新闻分类与新闻**：一个分类包含多个新闻，一个新闻属于一个分类（一对多关系）
6. **商品分类与商品**：一个分类包含多个商品，一个商品属于一个分类（一对多关系）
7. **用户与订单**：一个用户可以有多个订单，一个订单属于一个用户（一对多关系）
8. **订单与订单商品**：一个订单包含多个商品，一个商品属于一个订单（一对多关系）
9. **用户与购物车**：一个用户可以有多个购物车商品，一个购物车商品属于一个用户（一对多关系）
10. **用户与积分记录**：一个用户可以有多个积分记录，一个积分记录属于一个用户（一对多关系）
11. **用户与评论**：一个用户可以有多个评论，一个评论属于一个用户（一对多关系）

## 使用说明

### 1. 数据库创建
- 使用MySQL数据库管理系统
- 运行`zhihongfan_db.sql`文件创建数据库和表结构
- 数据库名称：`zhihongfan_db`
- 字符集：`utf8mb4`
- 排序规则：`utf8mb4_unicode_ci`

### 2. 初始化数据
- SQL文件中已经包含了初始化数据，包括：
  - 会员等级初始化数据
  - 照片画廊分类初始化数据
  - 新闻分类初始化数据
  - 商品分类初始化数据
  - 系统配置初始化数据

### 3. 数据库连接配置
在后端代码中配置数据库连接信息：
```javascript
// 示例：Node.js + MySQL连接配置
const mysql = require('mysql');

const connection = mysql.createConnection({
  host: 'localhost',
  user: 'root',
  password: 'your_password',
  database: 'zhihongfan_db',
  charset: 'utf8mb4'
});
```

### 4. 示例查询

#### 查询所有会员等级
```sql
SELECT * FROM member_levels ORDER BY min_points ASC;
```

#### 查询某个专辑的所有歌曲
```sql
SELECT * FROM songs WHERE album_id = 1 ORDER BY track_number ASC;
```

#### 查询用户的所有订单
```sql
SELECT o.*, GROUP_CONCAT(oi.product_title SEPARATOR ',') AS product_titles
FROM orders o
LEFT JOIN order_items oi ON o.id = oi.order_id
WHERE o.user_id = 1
GROUP BY o.id
ORDER BY o.created_at DESC;
```

#### 查询新闻的所有评论
```sql
SELECT c.*, u.username, u.avatar
FROM comments c
LEFT JOIN users u ON c.user_id = u.id
WHERE c.target_type = 'news' AND c.target_id = 1 AND c.status = 1
ORDER BY c.is_top DESC, c.created_at DESC;
```

## 注意事项

1. **密码安全**：用户密码在数据库中存储时必须进行加密处理，建议使用bcrypt等安全的加密算法

2. **文件上传安全**：上传的文件路径在数据库中存储时，必须进行安全验证，防止路径遍历攻击

3. **SQL注入防护**：在执行数据库查询时，必须使用参数化查询或预编译语句，防止SQL注入攻击

4. **XSS防护**：在存储用户输入的内容（如评论、新闻内容等）时，必须进行HTML转义处理，防止XSS攻击

5. **数据备份**：定期对数据库进行备份，确保数据安全

6. **性能优化**：根据实际使用情况，为常用查询添加索引，优化数据库性能
